# Python dependencies for the model training and export scripts
# (train_price_forecast_model.py and export_price_forecast_model.py).
#
# The data collection and lookup-table scripts rely only on the standard
# library; only the XGBoost training and ONNX export steps need third-party
# packages. Install them into a virtual environment before running either:
#
#   python3 -m venv .venv
#   .venv/bin/pip install -r requirements.txt
#   .venv/bin/python script/train_price_forecast_model.py
#   .venv/bin/python script/export_price_forecast_model.py
#
# Most versions are lower-bounded, not pinned: the scripts use stable,
# long-standing APIs. Note root_mean_squared_error requires scikit-learn >= 1.4.
# The exception is the ONNX export toolchain (xgboost and the three onnx*
# packages), which is pinned — see the notes on those entries below.

# Training (train_price_forecast_model.py):
#
# xgboost is pinned because the export step (below) converts the trained model
# by parsing XGBoost's tree dump, whose format can shift between versions; pinning
# keeps training and export on the version their conversion was verified against.
joblib>=1.3
pandas>=2.0
scikit-learn>=1.4
xgboost==3.3.0

# ONNX export and parity verification (export_price_forecast_model.py):
#
# onnxmltools converts the trained XGBoost booster to an ONNX graph, onnx
# stamps/serializes it, and onnxruntime runs it to confirm its predictions
# match the Python model. numpy ships as a scikit-learn/pandas dependency but
# is imported directly to build the verification batch, so it is listed here.
#
# Unlike the lower-bounded training dependencies above, the three ONNX packages
# are pinned to exact versions. onnxmltools converts by parsing XGBoost's
# internal tree-dump format, and that coupling is brittle across releases — a
# mismatched pair tends to fail the conversion with an opaque error. Pinning the
# versions this export was verified against keeps a reinstall reproducible; to
# upgrade, bump them deliberately and re-run the script, which re-verifies parity.
numpy>=1.24
onnx==1.22.0
onnxmltools==1.16.0
onnxruntime==1.27.0
